Philosophy

Five principles

Design decisions that shape every component.

01

One G-Brain per agent, per use case

Each agent owns its data. The Collector's G-Brain holds factual observations — what HealthKit reported, when, raw counts. The Coach's G-Brain holds interpretations — what it noticed, what it told the user, follow-up hypotheses. Neither agent can corrupt the other's record. To add a third agent, share its Pilot node ID — no shared infrastructure required.

02

Pilot is the only transport primitive

All inter-node communication is one thing: pilotctl send-message <target> --data <json>. The Pilot daemon delivers the JSON to the target's inbox. The receiver classifies by content shape — no virtual port routing at the application level. E2E encrypted, NAT-traversed, no port forwarding needed.

03

Health intelligence is retrieval, not a chatbot

A FastAPI server backed by 17 systematic reviews and meta-analyses returns ranked, citable intervention recommendations given alert values or a free-text query. Results are ranked by cosine similarity with ZeroEntropy reranking planned. Registered as an OpenClaw skill in both agent workspaces.

04

gstack-ios closes the development loop

Claude Code with the gstack-ios skill pack drives xcodebuild, simulators, signing, perf traces, and TestFlight uploads. The iOS development loop runs entirely in the agent — 13 skills covering every edge.

05

Distributed data, distributed compute

Each agent is an independent unit — its own OpenClaw workspace, its own Pilot identity, its own G-Brain, its own Docker container. They coordinate only via send-message. Scaling means adding a new container and registering its node ID. No shared database, no central coordinator.

End-to-End

Data flow

From wrist to warehouse to reasoning, fully traced.

1
HealthSyncManager polls HealthKit
Anchored queries ensure every sample is captured exactly once. 27 on-device models compute HRV stability, ACWR, sleep efficiency, burnout CUSUM, and more — entirely on-device, no server dependency.
2
Samples encoded → Pilot envelope (JSON)
Each envelope carries a batch UUID, sample UUIDs, version tag, and the raw + computed values. The iOS app is itself a Pilot node — pilot-swift is a precompiled Go daemon running inside the app sandbox.
3
PilotSyncTransport calls pilotctl send-message → Collector node
End-to-end encrypted tunnel. NAT-traversed — no port forwarding needed. Pilot handles identity (Ed25519), delivery confirmation, and reconnect. The envelope lands as a JSON file in the Collector's ~/.pilot/inbox/. isLikelyAvailable() gates the send on-device.
4
Collector dedupes by UUID, writes to DuckDB, acks back
inbox_watcher.py classifies the JSON by content shape — a message with a samples array goes to the ingest path. INSERT OR IGNORE on sample UUID deduplicates. DuckDB gives single-writer durability with MVCC reads. Ack JSON (accepted/duplicate/rejected UUIDs) sent back to iPhone via send-message.
5
iOS advances HK anchor only after Ack received
The HealthKit anchor is a write barrier. It only advances when the Collector has confirmed persistence. A crash or network failure causes the same samples to re-send — they are deduped on arrival by UUID.
6
Collector sends ChangeEvent to Coach via send-message
After each batch commit, change_event.py fires: {kind:"samples_added", table, new_count, since_ts}. Delivered to the Coach's Pilot inbox as a file. No virtual port routing — the Coach classifies it by the kind field.
7
Coach runs 7 rule models; sends SQL query to Collector
On each ChangeEvent, the Coach sends a JSON message with a sql field to the Collector via send-message. The Collector's sql_gate executes read-only queries and sends a QueryResult back. The Coach correlates the reply by request_id.
8
health-intelligence retrieves paper-backed interventions
Alert values trigger exact-match lookup in the interventions index. A free-text query runs cosine similarity over all-MiniLM-L6-v2 384-dim embeddings. Results include paper citations, score, and an LLM-ready prompt guardrailed to only cite indexed sources. Registered as an OpenClaw skill in both agent workspaces.
9
Coach writes insight to G-Brain; sends Telegram nudge
If a rule fires and cooldown has elapsed, the Coach writes an insight to its G-Brain (gbrain-coach-home) and sends a proactive nudge on Telegram via OpenClaw's channel binding. On-demand questions answered the same way — DuckDB + G-Brain recall + RAG evidence, ≤200 words, no pipe tables.

Networking

One primitive, content-based routing

Every message between nodes is delivered via pilotctl send-message <target-node-id> --data '<json>'. The daemon drops the JSON in the target's ~/.pilot/inbox/. inbox_watcher.py classifies by content shape — not by port number.

Content shape Classified as Action
samples array present HealthKit envelope Dedupe → write DuckDB → send Ack back to iPhone
sql field present SQL query from Coach sql_gate executes read-only → sends QueryResult back
kind: "samples_added" ChangeEvent Coach runs 7 rule models; may send proactive Telegram nudge
agent / command, no samples / sql Pilot overlay reply Passed to G-Brain ingester or heartbeat agent

Real Pilot built-in services: dataexchange port 1001 (used by send-message), echo port 7, handshake port 444. The reply_port: 1005 tag in query bodies is a correlation metadata field — not a Pilot routing port.

Retrieval

health-intelligence

Evidence-based intervention retrieval. Two paths. No hallucination — only indexed paper text returned.

alert_match

Exact alert lookup

Given a triggered alert with metric_id + band, returns all interventions indexed to that metric. Score fixed at 1.0. No embeddings. Sub-5ms response.

Trigger: alerts list in request · Speed: <5 ms
semantic

Cosine similarity retrieval

Free-text query embedded with all-MiniLM-L6-v2 (384-dim), L2-normalised, compared against cached paper embeddings. Skips texts already returned by alert path.

Trigger: query string · Speed: ~10 ms warm · Cold start: ~11 s (model load)
planned

ZeroEntropy reranking

Architecture reserves a reranking stage between retrieval and formatting. ZeroEntropy integration is planned — not yet implemented. Current fallback is cosine similarity order.

Requires: ZEROENTROPY_API_KEY in infra/.env

Infrastructure

Why Pilot Protocol

Every inter-component link uses Pilot. Here is why that matters.

🔐

End-to-end encrypted

Ed25519 identity per node. The iPhone, the Collector, and the Coach all have cryptographic identities. No shared secret, no bearer token.

🌐

NAT traversal built-in

No port forwarding. No VPN configuration. No firewall rules between the iPhone and the GCP VM. Pilot handles NAT traversal transparently — the tunnel works from any network.

🔄

Survive restarts independently

Either agent can restart without the other noticing beyond a brief reconnect. The trust is persistent on disk. Sessions resume automatically.

📦

Embedded in the iOS app

pilot-swift is a precompiled Go Pilot daemon distributed as a static xcframework. The iPhone is itself a first-class Pilot node — no separate hardware needed.

🎯

Delivery confirmation

The HealthKit anchor only advances after an Ack lands back from Agent A. Pilot guarantees delivery notification — the iOS app knows exactly what was accepted.

⚡

No routing table

No HTTP routing config, no internal DNS, no service mesh. Agents address each other by Pilot node ID. The overlay handles routing.

Security

Data sovereignty

The GCP VM is the trusted root. Each agent controls its own data — DuckDB volume and G-Brain instance are not shared. External services never see raw health samples.

Data chain

📱
iPhone collects
HealthKit samples stay on device until Pilot envelope is confirmed delivered to the Collector.
🔐
Pilot transports
E2E encrypted. Ed25519 identity per node. No data visible in transit — not even to the Pilot relay.
☁️
Collector warehouses in Docker volume
facts.duckdb lives in Docker volume docker_agent_a_data. Persists across container rebuilds. No external database.
🧠
Two isolated G-Brain instances
Each agent owns its own PGLite database — gbrain-collector-home for factual memory, gbrain-coach-home for interpretive memory. Neither can overwrite the other's record.
